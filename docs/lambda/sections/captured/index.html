    Captured variables #

    Terminology. A lambda expression may use variables that:

    • are declared outside of the lambda expression’s body, and
    • are not arguments of the lambda expression.

    In this case, the expression is called a capturing lambda expression, and these variable are said to be captured.

    Example. In the following Java program, the lambda expression x -> x == a is a capturing lambda, because it uses the variable a, which is defined outside of the lambda expression.

    boolean myMethod() {
    int a = 2;
    return execute(
            3,
            x -> x == a
    );
}

boolean execute(Integer x, Function<Integer, Boolean> function) {
    return function.apply(x);
}

    Restrictions #

    In practice, lambda expressions are often used to represent functions with no side effect.

    Some languages (like Java, JavaScript, C++, etc) restrict (or allow the user to restrict) the usage of lambda expressions accordingly.

    Captured variables in Java #

    Restriction. A capture variable in Java must be effectively final, meaning that it cannot be modified after its initialization.

    Examples. The following Java program is nearly identical to the previous one, but does not compile, because the captured variable a is not effectively final.

    boolean myMethod() {
    int a = 2;
    a = a + 1;
    return execute(
            3,
            x -> x == a
    );
}

boolean execute(Integer x, Function<Integer, Boolean> function) {
    return function.apply(x);
}

    The following program does not compile either, for the same reason.

    boolean myMethod() {
    int a = 2;
    return execute(
            3,
            x -> {
                a = a + 1;
                return x == a;
            }
    );
}

boolean execute(Integer x, Function<Integer, Boolean> function) {
    return function.apply(x);
}

    Exception. An exception to the rule above is captured variables that are allocated on the heap, i.e (class or instance) attributes.

    For instance, the following program is nearly identical to the (first) program above that does not compile. However, this new program compiles, because the captured variable a is now an instance attribute.

    public class EnclosingClass {

    int a = 2;

    boolean myMethod() {
        a = a + 1;
        return execute(
                3,
                x -> x == a
        );
    }

    boolean execute(Integer x, Function<Integer, Boolean> function) {
        return function.apply(x);
    }
}
